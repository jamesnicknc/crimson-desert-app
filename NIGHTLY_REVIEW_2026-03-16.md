## 🚨 Critical Issues Found

### Supabase RLS Infinite Recursion — group_members (FIXED)

Every page load was generating this console error:

```
[useProgress] Failed to load: infinite recursion detected in policy for relation "group_members"
```

Root cause: the "Members can view group peers" SELECT policy on `group_members` contained a subquery that referenced `group_members` itself, causing Postgres to recurse infinitely when evaluating the policy.

Fix applied via Supabase migration `fix_group_members_rls_recursion`:
- Created `get_my_group_ids()` SECURITY DEFINER function that bypasses RLS to fetch the current user's group memberships
- Dropped the recursive SELECT policy
- Recreated the policy using the SECURITY DEFINER function (no recursion)

Verified: zero console errors after reload. Fix is live.
